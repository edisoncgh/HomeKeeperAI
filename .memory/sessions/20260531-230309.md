# Session 2026-05-31 23:03:09

## Request

用户反馈创建管理员并登录后，访问 `/categories` 和 `/locations` 会反复跳转到登录页；要求使用 diagnose 定位修复，并同步 project-memory。

## Diagnosis

- HTTP 会话验证显示同一 Cookie 下 `/categories`、`/locations` 和 `/api/users/me` 均可返回 200，说明服务端认证路径本身可用。
- 新增回归测试模拟开发环境未配置 `AUTH_SECRET` 时的模块重载，发现旧 token 在重新 import `lib/auth/session.ts` 后失效。
- 根因：开发 fallback secret 原本是模块级随机值。Next dev 编译新路由或模块重载后会重新生成随机密钥，导致旧 Cookie 签名失效，受保护页面被判定未登录并跳 `/login`。

## Fix

- `lib/auth/session.ts` 改为把开发 fallback secret 存到 `globalThis.authDevSecret`，同一 dev 进程内保持稳定。
- `tests/auth-session.test.ts` 增加模块重载后 token 仍可验证的回归测试。
