# Session 20260602-200231

## Summary

诊断并修复 M3.4 拍照识别真实试用反馈：`/settings` 文本连接成功，但“识别候选”返回 502 且提示没有可用候选。

## Diagnosis

- 502 的错误分支位于 `lib/api/ai-recognition.ts`：LLM 返回内容可解析，但 `parseAiCandidateResponse` 得到 0 个有效候选。
- `/settings` 测试连接只发送文本 chat/completions，不验证当前模型是否支持 OpenAI 兼容 vision `image_url`。
- 常见兼容模型输出如 `source=vision/visual/photo` 或 `confidence=90` 会被原 parser 过滤，导致候选为空。
- 如果 LLM 返回 `candidates: []` 且 warnings 表示无法读取图片，则大概率是模型不支持 vision 或服务没有正确处理图片输入。

## Changes

- `lib/ai/prompts.ts`：拍照识别 Prompt 增加具体 JSON 示例，并强调 `confidence` 必须使用 0-1 小数。
- `lib/ai/parse.ts`：兼容 `source=vision/visual/photo` 并归一为 `image`；兼容 0-100 百分比置信度并归一为 0-1。
- `lib/api/ai-recognition.ts`：空候选时把脱敏 warnings 放入错误 message 和 `data.warnings`，方便用户区分格式过滤和模型不支持 vision。
- `tests/ai-contract.test.ts`、`tests/ai-candidate-parser.test.ts`、`tests/ai-recognition-api.test.ts`：新增回归覆盖。
- 同步 `docs/TASKS.md`、`docs/API.md`、`docs/QUALITY.md`、`docs/AGENT_HANDOFF.md` 和 `.memory/`。
