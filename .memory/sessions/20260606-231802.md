## Analysis
- 备份恢复优先保护 SQLite 数据库文件，默认目录应与 Compose 的 `/app/backups` Volume 对齐。
- 恢复是高风险操作，必须显式确认，并在覆盖数据库前创建当前数据库保护性备份。
- 备份 id 应使用严格文件名白名单，避免路径穿越或删除目录外文件。
- Docker/Linux 构建需要 builder 阶段也具备运行目录，否则服务端页面读取备份目录时可能在 page data 阶段触发构建异常。

## Actions
- 新增 `lib/backups/sqlite.ts`：解析 SQLite `DATABASE_URL`、定位 `BACKUP_DIR`、创建/列出/恢复/删除备份、校验备份文件名。
- 新增 `lib/api/backups.ts` 和 API route：`GET/POST /api/backups`、`POST /api/backups/restore`、`DELETE /api/backups/[id]`。
- 新增 `components/settings/backup-maintenance-panel.tsx` 并接入 `app/settings/page.tsx`。
- 新增 `tests/backup-storage.test.ts`、`tests/backup-api.test.ts`、`tests/settings-backup-ui.test.ts`。
- 更新 `.env.example`、`docker-compose.yml` 和 `Dockerfile`：新增 `BACKUP_DIR`，并补齐 builder 阶段 `/app/data`、`/app/uploads`、`/app/backups`。
- 修复旧的 `/api/backups/[id]/restore` 动态嵌套路由造成的 Docker/Linux 构建风险，改为静态 `POST /api/backups/restore`。
- 更新 `Readme.md`、`docs/TASKS.md`、`docs/API.md`、`docs/DEPLOYMENT.md`、`docs/TECH_PLAN.md`、`docs/QUALITY.md`、`docs/DECISIONS.md`、`docs/CONTEXT.md`、`docs/AGENT_HANDOFF.md` 和 `.memory`。
